Dataset Composition:
  • Total training samples: 2,000 (1,000 PostgreSQL + 1,000 MySQL)
  • Features per sample: 23 (15 scalar + 5 structural + 3 derived)
  • Query templates: 22 TPC-H templates × 2 databases
  • Training/Validation/Test split: 64% / 16% / 20%

Model Architecture:
  • Algorithm: LightGBM Gradient Boosting
  • Learning rate: 0.1
  • Max depth: 7
  • Num leaves: 31
  • Boost rounds: 200
  • Early stopping: 10 rounds with no improvement

Feature Engineering:
  • Cost-to-row ratio: total_cost / (actual_rows + 1)
  • Estimated-actual ratio: estimated_rows / (actual_rows + 1)
  • Time-cost ratio: actual_total_time / (total_cost + 1)

Preprocessing:
  • Numeric features: StandardScaler normalization
  • Categorical features: LabelEncoder
  • Missing values: None (synthetic data)

🎯 DEPLOYMENT OPTIONS
================================================================================

OPTION 1: Database-Specific Models (Best for Single-DB Systems)
  • Use postgresql_lightgbm.pkl for PostgreSQL databases
  • Use mysql_lightgbm.pkl for MySQL databases
  • Better accuracy per database
  • Requires conditional routing logic

OPTION 2: Unified Model (Best for Multi-Database Systems)
  • Use unified_lightgbm.pkl for any database type
  • Single deployment, handles both databases
  • Simpler implementation
  • Slight accuracy trade-off

OPTION 3: Ensemble Approach (Best for Critical Systems)
  • Deploy all three models
  • Use database type to select primary model
  • Fall back to unified model for unknown types
  • Maximum flexibility and reliability
